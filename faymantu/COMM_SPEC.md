## 1. 系統概述

MANPADS 追蹤系統提供最多 4 組裝置的即時 6DOF（6 自由度）追蹤資料，包含 3 軸位置與 3 軸旋轉，以及每組裝置的 8-bit IO 狀態。

系統提供兩種資料取得方式：

| 方式 | 通訊協議 | 適用場景 | 更新頻率 |
|------|----------|----------|----------|
| **WebSocket 推送** | WebSocket (RFC 6455) | 跨電腦 / 同電腦 | ~128 msg/sec |
| **本地檔案讀取** | 檔案 I/O | 僅同一台電腦 | ~500 次/sec（檔案覆寫） |

> 兩種方式**無法同時運行**，同一時間僅能選擇其中一種。

---

## 2. 裝置列表

| 裝置名稱 | Tag 代號 | 追蹤資料 | IO 模組 |
|----------|----------|----------|---------|
| 刺針（發射器） | `A` | 位置 / 旋轉 | 有（含訊號定義） |
| 頭盔 | `B` | 位置 / 旋轉 | 有 |
| 望遠鏡 | `C` | 位置 / 旋轉 | 有 |
| 對講機 | `D` | 位置 / 旋轉 | 有 |

- 裝置數量不固定（0 ~ 4），連線時自動出現，斷線時自動消失
- 以 `tag` 欄位識別裝置，陣列順序不保證

## 3. 通訊方式一：WebSocket

### 3.1 連線規格

| 項目 | 規格 |
|------|------|
| 通訊協定 | WebSocket (`ws://`) |
| 連線位址 | `ws://<Server IP>:<Port>` |
| 預設 Port | `8765` |
| Frame 類型 | Text frame |
| 編碼 | UTF-8 |
| 推送頻率 | ~128 msg/sec |
| 資料方向 | Server → Client（推送）；Client → Server（控制指令） |
| 最大 Client 數 | 無硬性上限（建議 ≤ 10） |
| 連線行為 | 連上後自動接收，不需發送任何請求 |

### 3.2 網路設定（由雙方協定後填入）

| 項目 | 設定值 |
|------|--------|
| Server IP | __________________ |
| Port | 8765 |
| 完整連線位址 | ws://_______________:8765 |
| 網路介面 | □ 有線 Ethernet　□ WiFi |
| SSID（WiFi） | __________________ |
| Password（WiFi） | __________________ |

### 3.3 WebSocket 資料格式

每一筆 WebSocket message 為一行 JSON 字串（無換行、無多餘空格）。

**原始接收字串範例（2 台裝置連線）：**


## 4. 通訊方式二：本地檔案

### 4.1 檔案規格

| 項目 | 規格 |
|------|------|
| 檔案路徑 | `build/Release/tracking_data.json`（相對於專案根目錄） |
| 寫入方式 | Atomic Write（先寫 `.tmp` → `rename` 替換） |
| 覆寫頻率 | ~500 次/sec（每 2ms） |
| 讀取建議 | Poll-read，間隔 5~10ms |
| 編碼 | UTF-8 |

### 4.2 本地檔案 JSON 格式

本地檔案使用 **C++ 完整格式**，比 WebSocket 格式多出 `scene`、`sceneName`、`id`、`type`、`number`、`stability`、全域 `io` 等欄位。

```json
{
  "scene": 1,
  "sceneName": "FV_10bars",
  "trackers": [
    {
      "id": 1,
      "tag": "A",
      "type": "A",
      "number": "1",
      "px": 1.256012,
      "py": 1.680032,
      "pz": 0.000145,
      "rx": -0.710743,
      "ry": 0.008621,
      "rz": -0.014023,
      "rw": 0.703312,
      "stability": 2,
      "io": "00000000"
    }
  ],
  "io": {
    "connected": true,
    "type": "A",
    "inputs": [0, 0, 0, 1, 0, 0, 0],
    "io7out": false
  }
}
```

### 4.3 兩種格式差異對照

| 欄位 | WebSocket 格式 | 本地檔案格式 |
|------|:-----------:|:-----------:|
| `trackers[].tag` | ✓ | ✓ |
| `trackers[].px` ~ `rw` | ✓ | ✓ |
| `trackers[].io` | ✓ | ✓ |
| `trackers[].id` | — | ✓ |
| `trackers[].type` | — | ✓ |
| `trackers[].number` | — | ✓ |
| `trackers[].stability` | — | ✓ |
| `scene` / `sceneName` | — | ✓ |
| `io`（全域） | — | ✓ |

> 建議：不論使用哪種方式，都以 `tag` 欄位識別裝置、以 `trackers[].io` 讀取 IO 狀態，這樣程式邏輯可在兩種模式間共用。

## 5. 欄位定義

### 5.1 trackers 陣列

JSON 根物件唯一必要的欄位（WebSocket 格式）。

| 欄位 | 型別 | 單位 | 範圍 | 說明 |
|------|------|------|------|------|
| `tag` | string | — | `"A"` `"B"` `"C"` `"D"` | 裝置代號（見第 2 節） |
| `px` | float | 公尺 (m) | 無限制 | X 軸位置（左右） |
| `py` | float | 公尺 (m) | 無限制 | Y 軸位置（高度），**Y 軸朝上** |
| `pz` | float | 公尺 (m) | 無限制 | Z 軸位置（前後） |
| `rx` | float | — | -1.0 ~ 1.0 | 旋轉四元數 X |
| `ry` | float | — | -1.0 ~ 1.0 | 旋轉四元數 Y |
| `rz` | float | — | -1.0 ~ 1.0 | 旋轉四元數 Z |
| `rw` | float | — | -1.0 ~ 1.0 | 旋轉四元數 W |
| `io` | string | — | 8 字元，每字元 `"0"` 或 `"1"` | IO 狀態（見第 6 節） |

**座標系**：右手座標系，Y 軸朝上。`py` 值即為裝置離地面的高度。

**旋轉**：四元數格式 (x, y, z, w)，滿足 x² + y² + z² + w² = 1。可轉換為 Euler 角：

| Euler 角 | 說明 |
|----------|------|
| Yaw | 偏航（左右轉） |
| Pitch | 俯仰（上下看） |
| Roll | 翻滾（歪頭） |

**精度**：浮點數為 6 位小數（`%.6f` 格式），約 0.001mm 的空間解析度。

### 5.2 本地檔案模式額外欄位

| 欄位 | 型別 | 說明 |
|------|------|------|
| `scene` | int | 當前場景編號（1-based） |
| `sceneName` | string | 場景名稱 |
| `trackers[].id` | int | Tracker 流水號（程式啟動後遞增） |
| `trackers[].type` | string | 裝置 Type 屬性（硬體設定值） |
| `trackers[].number` | string | 裝置 Number 屬性（硬體設定值） |
| `trackers[].stability` | int | 追蹤穩定度（見第 7 節） |
| `io.connected` | bool | 第一組 IO 模組是否已連接 |
| `io.type` | string | 第一組 IO 模組的 Type |
| `io.inputs[]` | int[7] | 7 路 Input 狀態（0 或 1） |
| `io.io7out` | bool | IO7 Output 狀態 |

### 5.3 各種連線狀態下的 JSON

| 狀態 | WebSocket 收到的 JSON |
|------|----------------------|
| 無裝置連線 | `{"trackers":[]}` |
| 僅刺針（A）連線 | `{"trackers":[{"tag":"A","px":...,"rw":...,"io":"00000000"}]}` |
| 刺針 + 頭盔 | `{"trackers":[{"tag":"A",...},{"tag":"B",...}]}` |
| 4 台全部連線 | `{"trackers":[{"tag":"A",...},{"tag":"B",...},{"tag":"C",...},{"tag":"D",...}]}` |
| 裝置無 IO 模組 | 該裝置的 `io` 欄位為 `"00000000"` |

## 6. IO 定義

### 6.1 IO 8-bit 字串格式

`io` 欄位為固定 **8 字元**字串，每個字元為 `"0"` 或 `"1"`。

| 字元位置 | 硬體腳位 | 方向 | `"0"` | `"1"` |
|----------|----------|------|-------|-------|
| `[0]` | IO1 | 輸入 | 無訊號 (High) | 有訊號 (Low) |
| `[1]` | IO2 | 輸入 | 無訊號 | 有訊號 |
| `[2]` | IOA3 | 輸入 | 無訊號 | 有訊號 |
| `[3]` | IOA4 | 輸入 | 無訊號 | 有訊號 |
| `[4]` | IO5 | 輸入 | 無訊號 | 有訊號 |
| `[5]` | IO6 | 輸入 | 無訊號 | 有訊號 |
| `[6]` | **IO7** | **輸出** | OFF (Low) | ON (High) |
| `[7]` | IO8 | 輸入 | 無訊號 | 有訊號 |

- 輸入腳位使用 **Active Low** 邏輯：硬體接地 (Low) = `"1"`（觸發），懸空 (High) = `"0"`（未觸發）
- IO7 為系統控制的輸出腳位（見第 8 節控制指令）
- 裝置無對應 IO 模組時，固定回傳 `"00000000"`

### 6.2 Tag A — 刺針 IO 定義

| 字元位置 | 硬體腳位 | 方向 | 訊號名稱 | `"0"` | `"1"` |
|----------|----------|------|----------|-------|-------|
| `[0]` | IO1 | 輸入 | **IFF 開關** | OFF | ON |
| `[1]` | IO2 | 輸入 | **鎖定鍵** | OFF | ON |
| `[2]` | IOA3 | 輸入 | **BCU** | OFF | ON |
| `[3]` | IOA4 | 輸入 | **保險** | OFF | ON |
| `[4]` | IO5 | 輸入 | **瞄準模組** | OFF | ON |
| `[5]` | IO6 | 輸入 | **板機** | OFF | ON |
| `[6]` | IO7 | **輸出** | **後座力** | OFF | ON |
| `[7]` | IO8 | 輸入 | （未定義） | — | — |

**解讀範例：**

| io 字串 | 狀態說明 |
|---------|----------|
| `"00000000"` | 所有訊號 OFF（待機狀態） |
| `"00110100"` | BCU=ON, 保險=ON, 瞄準模組=ON |
| `"11111000"` | IFF=ON, 鎖定=ON, BCU=ON, 保險=ON, 瞄準=ON |
| `"11111100"` | 上述全部 + 板機=ON（射擊中） |
| `"11111110"` | 上述全部 + 後座力=ON（射擊 + 後座力觸發中） |

## 7. 追蹤穩定度（僅本地檔案模式）

本地檔案模式的 `stability` 欄位表示追蹤精度等級：

| 值 | 狀態名稱 | 位置資料 | 旋轉資料 | 說明 |
|----|----------|:--------:|:--------:|------|
| 0 | 初始化中 | ✗ 不可用 | ✗ 不可用 | IMU 正在初始化 |
| 1 | 3DOF | **✗ 不準確** | ✓ 可用 | 僅 IMU 旋轉追蹤 |
| **2** | **6DOF** | **✓ 可用** | **✓ 可用** | **光學定位 + IMU（正常追蹤）** |
| 3 | 盲區 6DOF | △ 預測中 | ✓ 可用 | 暫時遮擋，靠慣性預測 |

> **重要**：`stability < 2` 時，`px`/`py`/`pz` 位置資料不準確，不應使用。**必須在場地中看見足夠燈柱，達到 `stability = 2` 後，位置資料才可信賴。**

> WebSocket 格式不包含 `stability` 欄位。若需判斷追蹤品質，可觀察位置資料是否穩定變化（`stability = 2` 時位置會正常移動，`stability = 1` 時位置幾乎不變）。

## 8. 控制指令（Client → Server）

### 8.1 IO7 控制（僅 WebSocket 模式）

Client 可透過 WebSocket 發送 JSON 指令，控制指定裝置的 IO7 輸出腳位。

**指令格式：**

```json
{"io7": "<Tag><State>"}
```

| 指令 | 目標裝置 | 動作 |
|------|----------|------|
| `{"io7":"A1"}` | Tag A（刺針） | IO7 → High (ON) |
| `{"io7":"A0"}` | Tag A（刺針） | IO7 → Low (OFF) |
| `{"io7":"B1"}` | Tag B（頭盔） | IO7 → High (ON) |
| `{"io7":"B0"}` | Tag B（頭盔） | IO7 → Low (OFF) |

### 8.2 注意事項

- IO7 在硬體端為 **toggle** 機制：收到指令後會切換一次 IO7 狀態
- **建議**：發送指令後，讀取下一筆回傳 JSON 的 `io[6]` 確認實際狀態
- 目前僅 Tag A（刺針）和 Tag B（頭盔）支援 IO7 控制
- 本地檔案模式不支援 IO7 控制指令（因無 WebSocket 通道）

### 8.3 確認 IO7 狀態

發送 IO7 指令後，從後續收到的 JSON 中確認：

```
回傳 JSON → trackers[tag="A"].io[6]
  "1" = IO7 ON (High)
  "0" = IO7 OFF (Low)
```

## 9. 效能規格

| 指標 | 數值 | 說明 |
|------|------|------|
| 追蹤硬體取樣率 | 2000 Hz | Antilatency IMU 內部 |
| C++ 主迴圈頻率 | 500 Hz | 每 2ms 輸出一次 |
| WebSocket 推送頻率 | ~128 msg/sec | 受 Windows pipe buffer 限制 |
| 本地檔案覆寫頻率 | ~500 次/sec | Atomic write |
| 每筆 JSON 大小 | ~150 - 400 bytes | 取決於連線裝置數量 |
| 延遲（同一台電腦） | < 10ms | WebSocket 或檔案 |
| 延遲（區域網路） | < 15ms | 有線 Ethernet 推薦 |
| 位置精度 | ~1mm | 依場地與燈柱數量 |
| 旋轉精度 | ~0.1° | IMU 融合 |

## 10. 啟動方式摘要

### WebSocket 模式

1. 追蹤主機雙擊 `faymantu/RunDataServer.bat`
2. 按 Enter 使用預設設定（Host: `0.0.0.0`、Port: `8765`）
3. Client 連線到 `ws://<Server IP>:8765`

### 本地檔案模式

1. 同一台電腦雙擊 `faymantu/RunLocalFile.bat`
2. 軟體組程式讀取 `build/Release/tracking_data.json`

### 連線測試

1. 雙擊 `faymantu/RunWSClient.bat`
2. 輸入 Server IP
3. 確認畫面顯示 JSON 資料且 RX ~128 msg/sec

> 詳細操作步驟、程式碼範例請見 `faymantu/SOFTWARE_GUIDE.md`。

## 11. 備註

- Client 連線後自動接收資料，**不需發送任何請求或握手指令**
- 裝置連線/斷線會即時反映在 `trackers` 陣列中
- 每個裝置的 IO 狀態獨立，透過 `tag` 欄位區分
- IO7 為輸出腳位（由追蹤系統或 Client 指令控制），其餘 7 個為輸入腳位
- 跨電腦使用時，需確保 Server 端防火牆已開放 TCP Port 8765
