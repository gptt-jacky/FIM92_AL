| 項目 | 內容 |
|------|------|
| 日期 | 2026-03-09 |
| 發行 | 追蹤組 → 費曼圖軟體組 |

---

## 系統架構

本系統由以下硬體組成：

- **Antilatency（AL）定位系統**：提供空間定位基礎設施，使用地墊系統
- **ALT 追蹤裝置（Alt Tracker）**：裝設於各訓練設備上，回傳即時位置與旋轉資料
- **AL IO 模組（HW Extension Module）**：連接各裝置的實體訊號腳位，讀取開關狀態並提供輸出控制

追蹤主機（Windows 電腦）透過 USB 接收 ALT 裝置與 IO 模組的原始資料，計算出 6DOF 定位結果（3 軸位置 + 3 軸旋轉），連同 IO 狀態整合為 JSON 格式，透過 WebSocket 即時推送給外部應用程式。

---

## 通訊規格

| 項目 | 規格 |
|------|------|
| 通訊協定 | WebSocket (`ws://`) |
| 預設 Port | `8765`（可在啟動時修改） |
| Frame 類型 | Text frame（UTF-8） |
| 資料格式 | JSON |
| 推送頻率 | ~128 msg/sec |
| 資料方向 | Server → Client（推送）；Client → Server（控制指令） |
| 連線行為 | 連上後自動接收，不需發送任何請求 |

## 裝置列表

| Tag | 裝置名稱 | 提供資料 |
|-----|----------|----------|
| `A` | 刺針本體 | 位置 / 旋轉 / IO |
| `B` | 主射手頭盔震動器 | 僅 IO（不提供位置資料） |
| `C` | 望遠鏡 | 位置 / 旋轉 / IO |
| `D` | 對講機 | 僅 IO（不提供位置資料） |
| `E` | 副射手頭盔 | 位置 / 旋轉 |
| `F` | 主射手頭盔 | 位置 / 旋轉 |

**注意事項：**

- 裝置數量不固定（0 ~ 6），連線時自動出現，斷線時自動消失
- 請用 `tag` 欄位識別裝置，**陣列順序不保證**
- Tag B/D 為 IO-only 裝置：位置與旋轉欄位固定為 0，僅 `io` 欄位有效
- Tag E/F 為 Alt-only 裝置：提供位置追蹤但無 IO，`io` 固定為 `"00000000"`

## 資料格式

每筆 WebSocket message 為一行 JSON 字串（無換行）。

**實際範例（6 台全連線）：**

```
{"trackers":[{"tag":"A","px":1.256012,"py":1.680032,"pz":0.000145,"rx":-0.710743,"ry":0.008621,"rz":-0.014023,"rw":0.703312,"io":"00110100"},{"tag":"B","px":0.000000,"py":0.000000,"pz":0.000000,"rx":0.000000,"ry":0.000000,"rz":0.000000,"rw":1.000000,"io":"00000000"},{"tag":"C","px":0.534210,"py":1.120045,"pz":-0.210034,"rx":-0.650123,"ry":0.012345,"rz":-0.008901,"rw":0.759012,"io":"10000000"},{"tag":"D","px":0.000000,"py":0.000000,"pz":0.000000,"rx":0.000000,"ry":0.000000,"rz":0.000000,"rw":1.000000,"io":"10000000"},{"tag":"E","px":0.912345,"py":1.750012,"pz":0.050678,"rx":-0.697612,"ry":0.008834,"rz":-0.010012,"rw":0.716023,"io":"00000000"},{"tag":"F","px":0.812000,"py":1.700056,"pz":0.060123,"rx":-0.701234,"ry":0.009012,"rz":-0.011234,"rw":0.712890,"io":"00000000"}]}
```

### 欄位定義

| 欄位 | 型別 | 說明 |
|------|------|------|
| `tag` | string | 裝置代號（`"A"` `"B"` `"C"` `"D"` `"E"` `"F"`） |
| `px` / `py` / `pz` | float | 位置（公尺）。Tag B/D 固定為 0 |
| `rx` / `ry` / `rz` / `rw` | float | 旋轉四元數 (x, y, z, w)。Tag B/D 固定為 (0, 0, 0, 1) |
| `io` | string | 8 字元 IO 狀態字串 |

---

## IO 字串定義

`io` 欄位為固定 **8 字元**字串，每個字元為 `"0"` 或 `"1"`。

### Tag A — 刺針本體

| 位置 | 腳位 | 方向 | 訊號名稱 | 說明 |
|------|------|------|----------|------|
| `[0]` | IO1 | 輸入 | IFF 開關 | 回報給軟體 |
| `[1]` | IO2 | 輸入 | 鎖定鍵 | **硬體觸發判定用** + 回報給軟體 |
| `[2]` | IOA3 | 輸入 | BCU | 回報給軟體 |
| `[3]` | IOA4 | 輸入 | 保險 | 回報給軟體 |
| `[4]` | IO5 | 輸入 | 瞄準模組 | 回報給軟體 |
| `[5]` | IO6 | 輸入 | 板機 | **硬體觸發判定用** + 回報給軟體 |
| `[6]` | IO7 | **輸出** | 允許後座力（軟體授權保險） | 費曼圖透過 WebSocket 控制 |
| `[7]` | IO8 | 輸入 | 後座力機構就緒 | 1=已就位可觸發 / 0=觸發後冷卻中 |

**IO7 語義說明：**

IO7 稱為「**允許後座力**」，作為軟體端授權保險，**不是直接觸發後座力**。後座力的實際觸發判定由硬體端本地執行。

- `{"io7":"A1"}` = **允許後座力**（軟體授權開啟，允許硬體端觸發後座力）
- `{"io7":"A0"}` = **禁止後座力**（軟體授權關閉，禁止硬體端觸發後座力）

硬體端觸發後座力需同時滿足三個條件：

| 條件 | 來源 | 說明 |
|------|------|------|
| IO7 = 1 | 軟體端 | 允許後座力（軟體授權保險已開） |
| IO8 = 1 | 硬體端 | 後座力機構已就位 |
| IO6 + IO2 持續 ≥ 1 秒 | 硬體端 | 板機 + 鎖定鍵同時按壓 |

**IO8 回報機制：**

- 後座力觸發後，IO8 **物理上自動歸 0**（機構動作，脫離就緒位置）
- 機構復位完成後，IO8 **自動回到 1**（約 10 秒）
- 軟體端只需讀取 `io[7]` 即可得知準備狀態，硬體端不需主動送訊號

**解讀範例：**

| io 字串 | 狀態說明 |
|---------|----------|
| `"00000000"` | 所有訊號 OFF（待機） |
| `"00110100"` | BCU=ON, 保險=ON, 瞄準模組=ON |
| `"11111100"` | IFF + 鎖定 + BCU + 保險 + 瞄準 + 板機（射擊中） |
| `"11111110"` | 上述全部 + IO7=ON（允許後座力） |
| `"11111101"` | 射擊中 + IO8=ON（後座力機構就緒） |

### Tag B — 主射手頭盔震動器

| 位置 | 腳位 | 方向 | 訊號名稱 |
|------|------|------|----------|
| `[0]`~`[5]` | IO1~IO6 | — | （未使用） |
| `[6]` | IO7 | **輸出** | 小震動 |
| `[7]` | IO8 | **輸出** | 大震動 |

**震動邏輯：**

| IO7 | IO8 | 效果 |
|-----|-----|------|
| OFF | — | 停止（IO8 單獨 ON 無效） |
| ON | OFF | 小震動 |
| ON | ON | 大震動 |

### Tag C — 望遠鏡

| 位置 | 腳位 | 方向 | 訊號名稱 |
|------|------|------|----------|
| `[0]` | IO1 | 輸入 | 縮小鍵 |
| `[1]` | IO2 | 輸入 | 放大鍵 |
| `[2]`~`[7]` | IO3~IO8 | — | （未使用） |

### Tag D — 對講機

| 位置 | 腳位 | 方向 | 訊號名稱 |
|------|------|------|----------|
| `[0]` | IO1 | 輸入 | 通話鍵 |
| `[1]`~`[7]` | IO2~IO8 | — | （未使用） |

Tag E（副射手頭盔）、Tag F（主射手頭盔）無 IO 模組，`io` 固定為 `"00000000"`。

## 控制指令（Client → Server）

### IO7 控制

```json
{"io7": "<Tag><State>"}
```

| 指令 | 語義 |
|------|------|
| `{"io7":"A1"}` | Tag A IO7 設為 ON（**允許後座力** — 軟體授權硬體端可觸發後座力） |
| `{"io7":"A0"}` | Tag A IO7 設為 OFF（**禁止後座力** — 軟體取消授權，硬體端不得觸發） |
| `{"io7":"B1"}` | Tag B IO7 設為 ON（**小震動**） |
| `{"io7":"B0"}` | Tag B IO7 設為 OFF（停止震動） |

> **Tag A IO7 語義（V1.0 定案，V1.1 介面統一命名）：** IO7 不直接觸發後座力，而是作為「允許後座力」的軟體授權訊號。IO7=1 期間，後座力觸發權交由硬體端根據 IO8（機構就位）+ IO6+IO2（板機+鎖定持續 1 秒）自行判定。軟體端任何時刻判定不可射擊，可立即送 `{"io7":"A0"}` 取消授權。

### IO8 控制（僅 Tag B）

```json
{"io8": "<Tag><State>"}
```

| 指令 | 語義 |
|------|------|
| `{"io8":"B1"}` | Tag B IO8 設為 ON（配合 IO7 一起 ON 為**大震動**；IO8 單獨 ON 無效） |
| `{"io8":"B0"}` | Tag B IO8 設為 OFF |

## Tag B 主射手頭盔震動器 — 操作對照

**場景一：待機 → 小震動**

- 發送：`{"io7":"B1"}`
- 結果：IO7=ON

**場景二：待機 → 大震動**

- 發送：`{"io7":"B1"}` + `{"io8":"B1"}`
- 結果：IO7=ON, IO8=ON

**場景三：小震動中 → 大震動**

- 發送：`{"io7":"B1"}` + `{"io8":"B1"}`
- 結果：IO7 不變, IO8=ON

**場景四：大震動 → 停止**

- 發送：`{"io7":"B0"}` + `{"io8":"B0"}`
- 結果：全部 OFF

**場景五：大震動 → 小震動**

- 發送：`{"io8":"B0"}`
- 結果：IO8=OFF, IO7 不變

### 注意事項

- **Set-to-state 語義**：指令中的 `1` / `0` 為目標狀態，Server 自動比對當前狀態，若已是目標狀態則不重複翻轉
- IO7 控制：僅 Tag A（刺針）和 Tag B（頭盔）支援
- IO8 控制：僅 Tag B（頭盔）支援

## IO 訊號監控工具（RunMonitor.bat）

提供 Console 即時 IO 訊號監控，測試訊號用。直接在追蹤主機上執行即可。

**啟動方式**：雙擊 `RunMonitor.bat`，Console 視窗會自動開始顯示。

**用途：**

- **訊號測試**：按下實體按鍵（板機、IFF 等），觀察對應 IO 是否即時變化
- **IO 輸出測試**：使用鍵盤快捷鍵（`A`/`B`/`O`）控制 IO7 輸出，確認硬體反應
- **追蹤狀態確認**：觀察 Stability 是否達到 `S:2`（6DOF 正常追蹤）
