## 1. 系統架構

本系統由以下硬體組成：

- **Antilatency（AL）定位系統**：提供空間定位基礎設施 使用地墊系統
- **ALT 追蹤裝置（Alt Tracker）**：裝設於各訓練設備上，回傳即時位置與旋轉資料
- **AL IO 模組（HW Extension Module）**：連接各裝置的實體訊號腳位，讀取開關狀態並提供輸出控制

追蹤主機（Windows 電腦）透過 USB 接收 ALT 裝置與 IO 模組的原始資料，計算出 6DOF 定位結果（3 軸位置 + 3 軸旋轉），連同 IO 狀態整合為 JSON 格式，透過 WebSocket 即時推送給外部應用程式。

---

## 2. 通訊規格

| 項目 | 規格 |
|------|------|
| 通訊協定 | WebSocket (`ws://`) |
| 預設 Port | `8765`（可在啟動時修改） |
| Frame 類型 | Text frame（UTF-8） |
| 資料格式 | JSON |
| 推送頻率 | ~128 msg/sec |
| 資料方向 | Server → Client（推送）；Client → Server（控制指令） |
| 連線行為 | 連上後自動接收，不需發送任何請求 |

---

## 3. 裝置列表

| Tag 代號 | 裝置名稱 | 提供資料 | 備註 |
|----------|----------|----------|------|
| `A` | 刺針（發射器） | 位置 / 旋轉 / IO | |
| `B` | 頭盔 | **僅 IO** | 不提供位置資料 |
| `C` | 望遠鏡 | 位置 / 旋轉 / IO | |
| `D` | 對講機 | **僅 IO** | 不提供位置資料 |

**注意事項：**

- 裝置數量不固定（0 ~ 4），連線時自動出現，斷線時自動消失
- 請用 `tag` 欄位識別裝置，**陣列順序不保證**
- Tag B、D 為 IO-only 裝置：位置與旋轉欄位固定為 0（`px=0, py=0, pz=0, rx=0, ry=0, rz=0, rw=1`），僅 `io` 欄位有效

## 4. 資料格式

每筆 WebSocket message 為一行 JSON 字串（無換行）。

**實際範例（刺針 A + 頭盔 B 同時連線）：**

```
{"trackers":[{"tag":"A","px":1.256012,"py":1.680032,"pz":0.000145,"rx":-0.710743,"ry":0.008621,"rz":-0.014023,"rw":0.703312,"io":"00000000"},{"tag":"B","px":0.000000,"py":0.000000,"pz":0.000000,"rx":0.000000,"ry":0.000000,"rz":0.000000,"rw":1.000000,"io":"00100000"}]}
```

**格式化後（方便閱讀）：**

```json
{
  "trackers": [
    {
      "tag": "A",
      "px": 1.256012,
      "py": 1.680032,
      "pz": 0.000145,
      "rx": -0.710743,
      "ry": 0.008621,
      "rz": -0.014023,
      "rw": 0.703312,
      "io": "00000000"
    },
    {
      "tag": "B",
      "px": 0.000000,
      "py": 0.000000,
      "pz": 0.000000,
      "rx": 0.000000,
      "ry": 0.000000,
      "rz": 0.000000,
      "rw": 1.000000,
      "io": "00100000"
    }
  ]
}
```

### 欄位定義

| 欄位 | 型別 | 說明 |
|------|------|------|
| `tag` | string | 裝置代號（`"A"` `"B"` `"C"` `"D"`） |
| `px` / `py` / `pz` | float | 位置（公尺），右手座標系，**Y 軸朝上**。Tag B/D 固定為 0 |
| `rx` / `ry` / `rz` / `rw` | float | 旋轉四元數 (x, y, z, w)。Tag B/D 固定為 (0, 0, 0, 1) |
| `io` | string | 8 字元 IO 狀態字串（見第 5 節） |

**各種連線狀態下的 JSON：**

| 狀態 | 收到的 JSON |
|------|------------|
| 無裝置連線 | `{"trackers":[]}` |
| 僅刺針（A）連線 | `{"trackers":[{"tag":"A","px":...,"rw":...,"io":"00000000"}]}` |
| 刺針 + 頭盔 | `{"trackers":[{"tag":"A",...},{"tag":"B",...}]}` |
| 4 台全連線 | `{"trackers":[{"tag":"A",...},{"tag":"B",...},{"tag":"C",...},{"tag":"D",...}]}` |

## 5. IO 字串定義

`io` 欄位為固定 **8 字元**字串，每個字元為 `"0"` 或 `"1"`。

### 5.1 IO 位置對照（通用）

| 字元位置 | 硬體腳位 | 方向 | `"0"` | `"1"` |
|----------|----------|------|-------|-------|
| `[0]` | IO1 | 輸入 | 無訊號 | 有訊號 |
| `[1]` | IO2 | 輸入 | 無訊號 | 有訊號 |
| `[2]` | IOA3 | 輸入 | 無訊號 | 有訊號 |
| `[3]` | IOA4 | 輸入 | 無訊號 | 有訊號 |
| `[4]` | IO5 | 輸入 | 無訊號 | 有訊號 |
| `[5]` | IO6 | 輸入 | 無訊號 | 有訊號 |
| `[6]` | **IO7** | **輸出** | OFF | ON |
| `[7]` | IO8 | 輸入 | 無訊號 | 有訊號 |

> **Tag B 例外**：IO8（`[7]`）也是輸出腳位（大震動）。

### 5.2 Tag A — 刺針（發射器）

| 位置 | 腳位 | 方向 | 訊號名稱 |
|------|------|------|----------|
| `[0]` | IO1 | 輸入 | IFF 開關 |
| `[1]` | IO2 | 輸入 | 鎖定鍵 |
| `[2]` | IOA3 | 輸入 | BCU |
| `[3]` | IOA4 | 輸入 | 保險 |
| `[4]` | IO5 | 輸入 | 瞄準模組 |
| `[5]` | IO6 | 輸入 | 板機 |
| `[6]` | IO7 | **輸出** | （未使用） |
| `[7]` | IO8 | 輸入 | 後座力準備 |

**解讀範例：**

| io 字串 | 狀態說明 |
|---------|----------|
| `"00000000"` | 所有訊號 OFF（待機） |
| `"00110100"` | BCU=ON, 保險=ON, 瞄準模組=ON |
| `"11111100"` | IFF + 鎖定 + BCU + 保險 + 瞄準 + 板機（射擊中） |
| `"11111110"` | 上述全部 + IO7=ON |

### 5.3 Tag B — 頭盔（IO-only）

| 位置 | 腳位 | 方向 | 訊號名稱 |
|------|------|------|----------|
| `[0]` | IO1 | 輸入 | （未使用） |
| `[1]` | IO2 | 輸入 | （未使用） |
| `[2]` | IOA3 | 輸入 | （未使用） |
| `[3]` | IOA4 | 輸入 | （未使用） |
| `[4]` | IO5 | 輸入 | （未使用） |
| `[5]` | IO6 | 輸入 | （未使用） |
| `[6]` | IO7 | **輸出** | 小震動 |
| `[7]` | IO8 | **輸出** | 大震動 |

**震動邏輯：**

| IO7 | IO8 | 效果 |
|-----|-----|------|
| ON | OFF | 小震動 |
| ON | ON | 大震動 |
| OFF | ON | 無效果 |
| OFF | OFF | 停止 |

### 5.4 Tag C — 望遠鏡

| 位置 | 腳位 | 方向 | 訊號名稱 |
|------|------|------|----------|
| `[0]` | IO1 | 輸入 | 縮小鍵 |
| `[1]` | IO2 | 輸入 | 放大鍵 |
| `[2]`~`[5]` | IOA3~IO6 | 輸入 | （未使用） |
| `[6]` | IO7 | **輸出** | （未使用） |
| `[7]` | IO8 | 輸入 | （未使用） |

### 5.5 Tag D — 對講機（IO-only）

| 位置 | 腳位 | 方向 | 訊號名稱 |
|------|------|------|----------|
| `[0]` | IO1 | 輸入 | 通話鍵 |
| `[1]`~`[5]` | IO2~IO6 | 輸入 | （未使用） |
| `[6]` | IO7 | **輸出** | （未使用） |
| `[7]` | IO8 | 輸入 | （未使用） |

## 6. 控制指令（Client → Server）

Client 可透過 WebSocket 發送 JSON 指令，設定裝置 IO 輸出狀態。

### 6.1 IO7 控制

```json
{"io7": "<Tag><State>"}
```

| 指令 | 目標裝置 | 語義 |
|------|----------|------|
| `{"io7":"A1"}` | Tag A（刺針） | IO7 → ON |
| `{"io7":"A0"}` | Tag A（刺針） | IO7 → OFF |
| `{"io7":"B1"}` | Tag B（頭盔） | IO7 → ON（小震動） |
| `{"io7":"B0"}` | Tag B（頭盔） | IO7 → OFF |

### 6.2 IO8 控制（僅 Tag B）

```json
{"io8": "<Tag><State>"}
```

| 指令 | 目標裝置 | 語義 |
|------|----------|------|
| `{"io8":"B1"}` | Tag B（頭盔） | IO8 → ON（大震動，需 IO7 同時 ON） |
| `{"io8":"B0"}` | Tag B（頭盔） | IO8 → OFF |

### 6.3 Tag B 頭盔震動操作對照

| 場景 | 發送指令 | 結果 |
|------|----------|------|
| 待機 → 小震動 | `{"io7":"B1"}` | IO7=ON |
| 待機 → 大震動 | `{"io7":"B1"}` + `{"io8":"B1"}` | IO7=ON, IO8=ON |
| 小震動中 → 大震動 | `{"io7":"B1"}` + `{"io8":"B1"}` | IO7 不變, IO8=ON |
| 大震動 → 停止 | `{"io7":"B0"}` + `{"io8":"B0"}` | 全部 OFF |
| 大震動 → 小震動 | `{"io8":"B0"}` | IO8=OFF, IO7 不變 |

### 6.4 注意事項

- **Set-to-state 語義**：指令中的 `1` / `0` 為目標狀態，Server 會自動比對當前狀態，若已是目標狀態則不會重複翻轉
- 建議發送指令後，讀取回傳 JSON 的 `io[6]`（IO7）、`io[7]`（IO8）確認實際狀態
- IO7 控制：僅 Tag A（刺針）和 Tag B（頭盔）支援
- IO8 控制：僅 Tag B（頭盔）支援

## 8. IO 訊號監控工具（RunMonitor.bat）

提供 Console 即時 IO 訊號監控，適合**硬體接線測試**與**訊號驗證**。不需 WebSocket 連線，直接在追蹤主機上執行即可。

### 8.1 啟動方式

雙擊 `RunMonitor.bat`，Console 視窗會自動開始顯示。

### 8.2 顯示畫面

```
======================================================================
  MANPADS Tracking Monitor    Scene: [1] FIM92    FPS: 128
======================================================================

  [A] 刺針  (T1, type=A)  S:2 Tracking
      Pos: (  +1.2560,   +1.6800,   +0.0001)
      Rot: (  -0.7107,   +0.0086,   -0.0140,   +0.7033)
      IO:  00110100   IO1=L  IO2=L  IOA3=H  IOA4=H  IO5=L  IO6=H  IO7*=L  IO8=L

  [B] 頭盔  (T0, type=B)  S:0 Init
      Pos: (  +0.0000,   +0.0000,   +0.0000)
      Rot: (  +0.0000,   +0.0000,   +0.0000,   +1.0000)
      IO:  00000000   IO1=L  IO2=L  IOA3=L  IOA4=L  IO5=L  IO6=L  IO7*=L  IO8=L

  ──────────────────────────────────────────────────────────────────
  Global IO [A]: 00110100   IO1=L  IO2=L  IOA3=H  IOA4=H  IO5=L  IO6=H  IO7*=L  IO8=L

  Frame #1600  |  [A] IO7-A  [B] IO7-B  [O] IO7-All  [1-9] Scene  [Q] Quit
```

### 8.3 顯示說明

| 區域 | 內容 |
|------|------|
| 標題列 | 當前場景名稱、資料接收速率 (FPS) |
| 各裝置區塊 | Tag 代號、裝置名稱、位置 (m)、旋轉 (Quaternion)、IO 8-bit 狀態 |
| IO 狀態 | 每個腳位標籤 + `H`=有訊號 / `L`=無訊號，IO7 標註 `*` 表示輸出腳位 |
| 追蹤穩定度 | `S:0` Init → `S:1` 3DOF → `S:2` Tracking（正常） → `S:3` Blind |
| Global IO | 第一組 HW Extension 的 IO 摘要（向下相容） |

### 8.4 用途

- **硬體接線驗證**：確認各裝置的 IO 腳位是否正確接通
- **訊號測試**：按下實體按鍵（板機、IFF 等），觀察對應 IO 是否即時變化
- **IO 輸出測試**：使用鍵盤快捷鍵（`A`/`B`/`O`）控制 IO7 輸出，確認硬體反應
- **追蹤狀態確認**：觀察 Stability 是否達到 `S:2`（6DOF 正常追蹤）

> 此工具僅在追蹤主機本地顯示，不提供 WebSocket 連線。若需將資料推送給外部程式，請使用 `RunDataServer.bat`。

## 9. 備註

- Client 連線後自動接收資料，**不需發送任何握手指令**
- 無裝置連線時收到 `{"trackers":[]}`
- Tag B / D 為 IO-only 裝置，位置欄位固定為 0，**請勿使用其位置資料**
- IO7 為輸出腳位；**Tag B 的 IO8 亦為輸出腳位**，其餘腳位均為輸入
- 跨電腦使用需確保 Server 防火牆開放 TCP Port 8765
- 詳細操作說明與連線範例請見 `faymantu/SOFTWARE_GUIDE.md`
