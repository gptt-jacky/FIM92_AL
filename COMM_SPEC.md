# MANPADS 追蹤系統通訊規格書

---

## 通訊規格

系統採用 WebSocket text frame 規格，由追蹤主機作為 WebSocket Server，持續推送即時追蹤資料（位置、旋轉、IO 狀態）給所有連線的 Client。

Client 連線後自動開始接收資料，不需發送任何指令。資料格式為 JSON，每筆 message 為一行完整 JSON 字串。Server 支援多個 Client 同時連線，所有 Client 接收到相同資料。

## 資料格式

### 原始接收字串

每一筆 WebSocket message 為一行 JSON 字串（無換行），以下為兩台裝置同時連線的範例：

```
{"trackers":[{"tag":"A","px":1.256012,"py":1.680032,"pz":0.000145,"rx":-0.710743,"ry":0.008621,"rz":-0.014023,"rw":0.703312,"io":"00000000"},{"tag":"B","px":0.500123,"py":1.500045,"pz":0.100012,"rx":-0.697612,"ry":0.008834,"rz":-0.010012,"rw":0.710023,"io":"00100000"}]}
```

### 格式化後（方便閱讀）

```json
{
  "trackers": [
    {
      "tag": "A",
      "px": 1.256012,
      "py": 1.680032,
      "pz": 0.000145,
      "rx": -0.710743,
      "ry": 0.008621,
      "rz": -0.014023,
      "rw": 0.703312,
      "io": "00000000"
    },
    {
      "tag": "B",
      "px": 0.500123,
      "py": 1.500045,
      "pz": 0.100012,
      "rx": -0.697612,
      "ry": 0.008834,
      "rz": -0.010012,
      "rw": 0.710023,
      "io": "00100000"
    }
  ]
}
```

### 欄位定義

| 欄位 | 型別 | 說明 |
|------|------|------|
| `trackers` | array | 所有已連線裝置的陣列（無裝置時為空 `[]`） |
| `tag` | string | 裝置代號（見裝置列表） |
| `px` | float | X 軸位置（單位：公尺） |
| `py` | float | Y 軸位置（單位：公尺，Y 軸朝上） |
| `pz` | float | Z 軸位置（單位：公尺） |
| `rx` | float | 旋轉四元數 X |
| `ry` | float | 旋轉四元數 Y |
| `rz` | float | 旋轉四元數 Z |
| `rw` | float | 旋轉四元數 W |
| `io` | string | 8 位元 IO 狀態字串（見下方 IO 定義） |

### 特殊情況

| 情況 | 收到的 JSON |
|------|-------------|
| 無裝置連線 | `{"trackers":[]}` |
| 僅 1 台裝置 | `{"trackers":[{"tag":"A",...}]}` |
| 4 台裝置全連 | `{"trackers":[{"tag":"A",...},{"tag":"B",...},{"tag":"C",...},{"tag":"D",...}]}` |
| 裝置無 IO 模組 | 該裝置的 `io` 為 `"00000000"` |

## IO 定義

`io` 欄位為固定 **8 字元**字串，每個字元（`"0"` 或 `"1"`）代表一個 IO 腳位。

- `"0"` = Low（無訊號）
- `"1"` = High（有訊號）
- 裝置無對應 IO 模組時固定回傳 `"00000000"`

### IO 腳位對照

| 字元位置 | 硬體腳位 | 方向 |
|----------|----------|------|
| `[0]` | IO1 | 輸入 |
| `[1]` | IO2 | 輸入 |
| `[2]` | IOA3 | 輸入 |
| `[3]` | IOA4 | 輸入 |
| `[4]` | IO5 | 輸入 |
| `[5]` | IO6 | 輸入 |
| `[6]` | IO7 | **輸出** |
| `[7]` | IO8 | 輸入 |

| 字元位置 | 硬體腳位 | 方向 | 訊號定義 | 說明 |
|----------|----------|------|----------|------|
| `[0]` | IO1 | 輸入 | IFF 開關 | `1`=ON, `0`=OFF |
| `[1]` | IO2 | 輸入 | 鎖定鍵 | `1`=ON, `0`=OFF |
| `[2]` | IOA3 | 輸入 | BCU | `1`=ON, `0`=OFF |
| `[3]` | IOA4 | 輸入 | 保險 | `1`=ON, `0`=OFF |
| `[4]` | IO5 | 輸入 | 瞄準模組 | `1`=ON, `0`=OFF |
| `[5]` | IO6 | 輸入 | 板機 | `1`=ON, `0`=OFF |
| `[6]` | IO7 | **輸出** | 後座力 | Server 控制輸出 |

**範例：** `"00110100"` → BCU=ON, 保險=ON, 瞄準模組=ON，其餘 OFF

## Server 端啟動說明

### 環境需求

| 項目 | 需求 |
|------|------|
| 作業系統 | Windows 10/11 (64-bit) |
| Python | 3.8 以上，已加入系統 PATH |
| Python 套件 | `pip install websockets` |
| 硬體 | Antilatency Alt Tracker + Extension Module（USB 連接至 Server 電腦） |
| 編譯產物 | `build/Release/TrackingMinimalDemo.exe`（需先編譯或從 Release 下載） |

### 啟動步驟

**Step 1：** 雙擊 `RunDataServer.bat`

會開啟一個 Console 視窗（標題為 `MANPADS_DataServer`），顯示以下畫面：

```
=== MANPADS Data Server (for Software Team) ===

Server Host [0.0.0.0]: _
```

**Step 2：** 輸入 Server Host（或直接按 Enter 使用預設值）

| 輸入內容 | 效果 |
|----------|------|
| 直接按 **Enter** | 使用預設 `0.0.0.0`（接受來自任何 IP 的連線，**LAN 使用推薦**） |
| 本機內網 IP（例如 `192.168.1.100`） | 僅綁定該網卡 IP，Client 需用此 IP 連線 |
| `127.0.0.1` | 僅允許本機連線（本機測試用） |

> **如何查看本機 IP：** 在 Server 電腦上開啟 CMD，執行 `ipconfig`，找到乙太網路或 Wi-Fi 的 **IPv4 位址**（例如 `192.168.1.100`），即可輸入作為 Host。

**Step 3：** 輸入 Server Port（或直接按 Enter 使用預設值）

```
Server Port [8765]: _
```

| 輸入內容 | 效果 |
|----------|------|
| 直接按 **Enter** | 使用預設 `8765` |
| 其他數字 | 使用自訂 Port（需確保防火牆允許） |

**Step 4：** Server 啟動完成

出現以下畫面表示已開始運行：

```
[Host] 0.0.0.0
[Port] 8765
[URI]  ws://0.0.0.0:8765

Starting...

  WebSocket server started on ws://0.0.0.0:8765
  [RX] 487/sec  [TX] 485/sec  |  Clients: 0
```

| 指標 | 說明 |
|------|------|
| RX | 從 C++ 追蹤程式收到的原始資料速率 |
| TX | 實際推送給 WebSocket Client 的速率 |
| Clients | 目前連線中的 Client 數量 |

> **注意：** Server 執行期間請勿關閉此 Console 視窗，關閉即停止服務。

### Client 端連線方式

Server 啟動後，Client 使用以下位址連線：

```
ws://<Server 的內網 IP>:<Port>
```

**範例：** 若 Server 電腦的內網 IP 為 `192.168.1.100`，Port 為預設 `8765`：

```
ws://192.168.1.100:8765
```

> 查看 Server 內網 IP：在 Server 電腦上執行 `ipconfig`，找到乙太網路或 Wi-Fi 的 IPv4 位址。

### 連線測試工具

可用 `RunWSClient.bat` 快速驗證連線：

```
=== MANPADS WebSocket Client ===

Server Host [localhost]: 192.168.1.100
Server Port [8765]: 8765
```

輸入 Server 的內網 IP 即可連線，會顯示即時收到的原始 JSON 和接收速率。
支援鍵盤快捷鍵：按 `A` / `B` 鍵即可 toggle 對應裝置的 IO7。

### 防火牆設定

跨機器使用時，需確保 Server 端防火牆允許 WebSocket Port 的 **TCP 入站**連線：

```powershell
netsh advfirewall firewall add rule name="MANPADS DataServer" dir=in action=allow protocol=tcp localport=8765
```

## 控制指令（Client → Server）

Client 可透過 WebSocket 發送 JSON 指令控制裝置 IO7 輸出。

### IO7 控制

| 指令 | 說明 |
|------|------|
| `{"io7":"A1"}` | Tag A IO7 → High |
| `{"io7":"A0"}` | Tag A IO7 → Low |
| `{"io7":"B1"}` | Tag B IO7 → High |
| `{"io7":"B0"}` | Tag B IO7 → Low |

注意：IO7 採 toggle 機制，Server 收到指令後會對該裝置的 IO7 做一次 toggle。建議搭配回傳的 `io` 欄位確認實際狀態。

## 備註

- Client 連線後自動接收資料，不需發送任何請求或握手指令
- 裝置連線 / 斷線會即時反映在 `trackers` 陣列中（有連線就有，斷線就消失）
- 每個裝置的 IO 狀態獨立，透過 `tag` 欄位區分
- IO7 為輸出腳位，由追蹤系統控制，其餘為輸入腳位
